# tint-win

[English](README.md)

一个很小的 Win32 小东西，用来把别的窗口变透明。

这个项目主要是我拿来练 C 和 Windows API 的。它会列出桌面上可见的普通窗口，让你选一个窗口，调一下透明度，然后在你不想玩了的时候尽量把窗口恢复回去。

差不多就这些。程序就这么大。

## 构建

需要 MSVC / Visual Studio Build Tools。

打开 Developer Command Prompt，或者任何能直接使用 `cl` 的命令行，然后运行：

```bat
build.bat
```

生成的 exe 在这里：

```text
build\tint.exe
```

## 它能做什么

- 列出普通的可见桌面窗口
- 显示当前选中窗口的标题和进程名
- 用滑块调整窗口透明度
- 恢复当前窗口
- 恢复这个程序改过的所有窗口
- 退出时尽量恢复被改过的窗口

## 它不能做什么

很多。

这不是一个想装得很完整的桌面软件。它就是一个 C/Win32 练手项目，只是刚好有点小用处，所以可以留着。

没有安装器，没有托盘图标，没有配置文件，没有自动更新，也没有什么花活。以后也许会有。也许永远没有。

## 代码大概长这样

现在代码被拆成了几个小块：

- `tint.c` - Win32 入口和消息处理
- `tint_app.h` - 共享的 app 状态和控件 id
- `tint_window_list.*` - 找窗口
- `tint_opacity.*` - 应用和恢复透明度
- `tint_ui.*` - 创建控件
- `tint_view.*` - 把文字和状态显示到界面上

## 一些说明

写这个主要是因为我可能有点想和 Win32 打一架。

Win32 API 给我的感觉很古老，像恐龙时代的化石突然摆在我眼前。微软官方文档也
很有意思，很多时候读起来像甲骨文，感觉答案就在上面，但翻译器没一起出土。

底层用的是 layered window attributes。有些窗口可能不会按你想的方式表现，因为 Windows 桌面程序就是 Windows 桌面程序。

如果哪里看起来不对，先点 restore。

如果 restore 也救不了，那至少我们又学到了一点东西。
